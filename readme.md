# 円判定ミニゲーム（Game Math Mini Game）

## 概要

本作品は **C++ / DxLib** を用いて制作した、
**ゲームに必要な数学（ベクトル・距離・円衝突判定）の理解を目的としたミニゲーム**です。

「数学・物理の知識が、実際のゲームロジックとしてどのように使われるのか」を
**最小構成のゲーム**として体験・説明できることを目標に制作しました。

---

## ゲーム内容

* プレイヤーは **円** として扱われます
* WASDキーで移動します
* 開始から **2秒後にプレイヤーは非表示** になります
* 障害物（固定円）や画面外に触れると **ゲームオーバー**
* ゴール円に到達すると **ゲームクリア**
* クリアまでの時間をスコアとして表示します

> 視覚情報に頼れない状態でも、
> 距離感・位置関係・判定を意識した操作が求められる設計です。

---

## 操作方法

| キー            | 動作      |
| ------------- | ------- |
| W / A / S / D | プレイヤー移動 |

## 実装した主な数学・ゲーム技術

### 1. 2Dベクトル（Vec2）

* 加算・減算
* 長さ（ノルム）
* 距離計算の基礎

```text
位置差ベクトル = A.pos - B.pos
```

---

### 2. 円同士の衝突判定

円同士の当たり判定は、
**中心間の距離が半径の合計以下かどうか**で判定しています。

```text
|A - B| ≤ rA + rB
```

平方根計算を避けるため、距離²で比較しています。

---

### 3. 障害物管理（vector）

* 障害物は `vector<Circle>` で管理
* 描画・判定をループで共通化
* コースは規則的に配置

```text
for each obstacle:
  if Hit(player, obstacle): GameOver
```

---

### 4. ゲームステート管理

```cpp
enum class GameState
{
    Title,
    Playing,
    GameOver,
    Clear
};
```

* 状態に応じて入力・更新・描画を制御
* 不正な入力を防止

---

### 5. 画面外判定

プレイヤーが画面外に出た場合、
非表示状態では復帰が困難になるため、
**即ゲームオーバー**としています。

```text
0 ≤ x ± r ≤ ScreenWidth
0 ≤ y ± r ≤ ScreenHeight
```

---

## 設計上の工夫・意図

* プレイヤー・敵・ゴール・障害物を
  **すべて円として統一**
* 判定ロジックを完全に共通化
* プレイヤー非表示により、
  数学的な距離感覚をゲーム体験に直結
* 再利用可能な数学・判定処理は
  ヘッダ・cppに分離
